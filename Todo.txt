Problem : Consumer gets event but DB has no data etc .
Solution : OutBox Pattern (Don’t publish events directly — store them first, then publish reliably.)

-> Since database and message queue are separate systems ,we cannot have a single transaction across them .
So we use the Transactional Outbox Pattern, where events are stored in the database and later published using a scheduler,
ensuring reliability and consistency.
